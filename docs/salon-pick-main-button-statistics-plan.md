# 살롱픽 메인버튼 통계 노출 계획

## 목표

상품 관리 페이지에서 어드민 관리자가 살롱픽 메인 진입 버튼의 클릭 흐름을 상품 슬롯 클릭수와 함께 확인할 수 있게 한다. 현재 구현은 API, 데이터 모델, 상품 관리 페이지 상단의 메인 버튼 통계 요약 영역까지 포함한다.

## 추가된 데이터 사용 방식

- 리스트 API: `GET /api/v1/admins/salon-pick-main-buttons`
  - 상품 관리 페이지 진입 시 첫 화면 요약에 필요한 데이터를 가져온다.
  - 기본 정렬은 `idDesc`, 기본 개수는 20개로 둔다.
  - `__nextCursor`가 있으면 향후 "더 보기" 또는 페이지네이션을 붙일 수 있게 유지한다.
- 상세 API: `GET /api/v1/admins/salon-pick-main-buttons/:salonPickMainButtonId`
  - 특정 버튼의 일별 클릭수 히스토리를 볼 때만 호출한다.
  - 페이지 최초 진입 때 상세까지 모두 부르지 않고, 상세 영역을 열거나 별도 탭에 진입할 때 호출하는 편이 낫다.

## 관리자 화면 배치 제안

상품 관리 페이지 상단 안내 문구 아래, 슬롯 테이블 위에 `메인 버튼 통계` 요약 영역을 둔다. 어드민은 상품 슬롯 클릭수를 이미 같은 페이지에서 보고 있으므로, 메인 버튼 클릭수도 같은 흐름 안에서 봐야 메인 진입 대비 상품 클릭 반응을 빠르게 비교할 수 있다.

요약 영역에는 다음 값을 먼저 노출한다.

- 버튼 코드: `MAIN_BUTTON` 같은 원본 코드와 사람이 읽기 쉬운 라벨을 함께 표시
- 전체 클릭수: `clickCount`
- 전일 클릭수: `yesterdayDailyClickCount.dailyClickCount`, 값이 없거나 `null`이면 0으로 표시
- 마지막 갱신 시각: `updatedAt`

상세 히스토리는 요약 영역 안에 항상 펼치기보다, "일별 내역" 액션으로 열리는 영역에 둔다. 상품 관리 페이지는 슬롯 생성, 활성화, 삭제가 주요 업무이므로 통계 상세가 기본 테이블 업무를 밀어내지 않게 해야 한다.

## 일별 내역 표현 원칙

- 상세 API의 `salonPickProductCounts`는 API 응답명과 달리 메인버튼의 일별 클릭수로 정규화해서 사용한다.
- 화면에서는 `dailyClickCounts`라는 이름으로만 다룬다.
- API는 클릭수가 있는 날짜만 내려주므로, 누락된 날짜를 화면에서 자동으로 0으로 채워 넣지 않는다.
- 날짜는 `YYYY.MM.DD`, 클릭수는 `ko-KR` 천 단위 포맷으로 표시한다.
- 최근순 정렬을 기본값으로 둔다.

## 상태 처리

- 로딩: 요약 영역 높이를 유지하는 스켈레톤 또는 "불러오는 중..." 상태를 둔다.
- 빈 데이터: "메인 버튼 통계가 없습니다."로 표시하고 상품 슬롯 테이블 사용은 막지 않는다.
- 에러: 요약 영역 안에서 재시도 버튼을 제공한다.
- 새로고침: 메인 버튼 통계 영역 안에서 별도로 재조회할 수 있게 한다.

## 구현 순서

1. 상품 관리 페이지 콘텐츠에서 `useGetSalonPickMainButtonsQuery`를 호출한다.
2. 리스트 응답의 각 버튼을 행 단위로 표시한다.
3. 버튼 코드가 여러 개가 될 가능성을 고려해 요약 컴포넌트 props를 배열 기반으로 설계한다.
4. "일별 내역"이 열릴 때 `useGetSalonPickMainButtonDetailQuery`를 활성화한다.
5. 기존 상품 슬롯의 전체 클릭수/전일 클릭수와 같은 포맷 유틸을 공유하지 않고, 메인버튼 전용 유틸을 사용해 도메인 이름이 섞이지 않게 한다.

## 확인 기준

- 상품 관리 페이지에서 상품 슬롯 데이터와 메인 버튼 통계가 서로 독립적으로 로딩, 실패, 재시도 상태를 처리한다.
- API 응답의 `salonPickProductCounts` 필드는 화면 코드에서 직접 사용하지 않는다.
- `yesterdaySalonPickProductCount`가 `null`이어도 전일 클릭수는 0으로 안정적으로 표시된다.
- 상세 히스토리 호출 실패가 상품 슬롯 관리 기능을 막지 않는다.
